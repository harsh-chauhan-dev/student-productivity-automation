# 🔔 Notification Management Module

The Notification Management Module provides an in-app notification system that keeps users informed about important events such as task reminders, study activities, and future system updates.

Notifications are automatically generated by backend services (such as the Reminder Automation Job) and can be viewed, marked as read, or deleted by the authenticated user.

---

# Overview

Every notification belongs to exactly one authenticated user.

Example

```text
Reminder Job

↓

Task Reminder Generated

↓

Notification Created

↓

Notification Center

↓

User Reads Notification
```

The notification system acts as the application's internal messaging center.

# Database Schema

## notifications

| Column     | Type      | Description          |
| ---------- | --------- | -------------------- |
| id         | UUID      | Primary Key          |
| user_id    | UUID      | Notification Owner   |
| title      | VARCHAR   | Notification Title   |
| message    | TEXT      | Notification Message |
| is_read    | BOOLEAN   | Read Status          |
| created_at | TIMESTAMP | Notification Time    |

# Authentication

All Notification APIs require authentication.

Authenticated user

```javascript
req.user.id
```

---

# API Endpoints

| Method | Endpoint                      | Description               |
| ------ | ----------------------------- | ------------------------- |
| GET    | `/api/notifications`          | Get All Notifications     |
| GET    | `/api/notifications/:id`      | Get Notification By ID    |
| PATCH  | `/api/notifications/:id/read` | Mark Notification as Read |
| DELETE | `/api/notifications/:id`      | Delete Notification       |

---

# Get All Notifications

## Request

```http
GET /api/notifications
```

### Success Response

```json
{
    "success": true,
    "count": 5,
    "notifications": []
}
```

Notifications are returned in descending order of creation.

---

# Get Notification By ID

## Request

```http
GET /api/notifications/:id
```

### Success Response

```json
{
    "success": true,
    "notification": {}
}
```

---

# Mark Notification as Read

## Request

```http
PATCH /api/notifications/:id/read
```

### Success Response

```json
{
    "success": true,
    "notification": {}
}
```

After this request

```text
is_read = true
```

# Delete Notification

## Request

```http

```

### Success Response

    "message": "Notification deleted successfully"

# Security

Every database query validates ownership.

Example

```sql
SELECT *

FROM notifications

WHERE id = $1

AND user_id = $2
```

This ensures users cannot view, modify, or delete notifications belonging to other users.
